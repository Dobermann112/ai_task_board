このプロジェクトではAIエージェントを開発補助として利用しています。

以下のルールを必ず守ってください。

## 基本方針

- まず現在のコードベースを調査してください。
- 既存の設計・命名規則・ディレクトリ構成を理解してから実装してください。
- 不要な設計変更は行わず、既存の実装方針を尊重してください。
- 実装前に今回の実装方針を説明してください。

## 実装ルール

- TypeScriptの型安全性を維持してください。
- 可読性を意識してください。
- コンポーネントは必要に応じて適切に分割してください。
- 将来的な保守性・拡張性を考慮してください。
- エラーハンドリングを実装してください。
- バリデーションを実装してください。
- コメントは必要最低限にしてください。

## README.md

README.mdはサービスの仕様書として扱います。

今回の変更によってREADMEへ追記・修正が必要な場合は必ず更新してください。

READMEには以下を記載してください。

- サービス概要
- 現在実装済み機能
- 技術スタック
- ディレクトリ構成（必要に応じて）
- データベース概要
- 今回追加した機能
- 起動方法
- 今後の実装予定（必要に応じて）

READMEは今後も機能追加のたびに最新状態へ更新してください。

---

## 動作確認

実装後は必ず以下を実施してください。

1.
npm run build

2.
npm run lint

3.
開発サーバーを起動してください。

npm run dev

4.
ブラウザで実際に画面を確認してください。

- 画面が正常に表示されるか
- コンソールエラーがないか
- 実装した機能が正常に動作するか
- UI崩れがないか
- 想定外の挙動がないか

ブラウザで確認した内容も報告してください。
